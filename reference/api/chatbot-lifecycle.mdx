Every MCC automation is a C# class that extends `ChatBot`, the abstract base class defined in `MinecraftClient/Scripting/ChatBot.cs`. Scripts are plain `.cs` files with a short metadata header; MCC compiles and loads them at runtime using Roslyn. Built-in bots bundled with MCC and standalone `/script` files share exactly the same event model and toolbox methods described on this page.

## Script file format

A standalone script must start with `//MCCScript 1.0` on the first line, followed by optional `//using` and `//dll` directives, then the bot instantiation passed to `MCC.LoadBot`, and finally `//MCCScript Extensions` before the class definition.

```csharp
//MCCScript 1.0

//using System.Collections.Immutable   // optional extra namespace
//dll MyHelpers.dll                    // optional extra DLL

MCC.LoadBot(new MyBot());

//MCCScript Extensions

public class MyBot : ChatBot
{
    public override void Initialize()
    {
        LogToConsole("Bot loaded!");
    }
}
```

Run a script with the MCC internal command `/script MyBot.cs`. The following namespaces are available without any `//using` directive: `System`, `System.Collections.Generic`, `System.Text.RegularExpressions`, `System.Linq`, `System.Text`, `System.IO`, `System.Net`, `System.Threading`, `MinecraftClient`, `MinecraftClient.Mapping`, and `MinecraftClient.Inventory`.

<Note>
Do **not** send chat or commands from `Initialize()`. The login handshake is not complete at that point. Use `AfterGameJoined()` instead.
</Note>

## Lifecycle events

These `virtual` methods are called by MCC at well-defined points. Override only the ones you need; every method has an empty default implementation so unhandled events are silently ignored.

### Initialize

```csharp
public virtual void Initialize()
```

Called exactly once when the bot is first loaded. Use this method to set up fields, caches, and other one-time state. Do not call `SendText` here.

---

### OnUnload

```csharp
public virtual void OnUnload()
```

Called when the bot is being unloaded. Release resources such as database connections, background threads, plugin channel registrations, and movement locks here.

---

### AfterGameJoined

```csharp
public virtual void AfterGameJoined()
```

Called after successfully joining a server and after every reconnect. This is the earliest safe point to call `SendText`. Unlike `Initialize`, this method fires again on each reconnect.

---

### Update

```csharp
public virtual void Update()
```

Called every client tick, approximately every 50 ms at 20 TPS. Suitable for lightweight polling loops. Avoid blocking operations; long-running work should be offloaded with `ScheduleOnMainThread` or a background `Task`.

---

### GetText

```csharp
public virtual void GetText(string text)
public virtual void GetText(string text, string? json)
```

Called for every text message sent by the server, including chat, system messages, and action bar text. The two-parameter overload also receives the raw JSON from the server; `json` is `null` on MC 1.5 and older.

Always strip color codes with `GetVerbatim(text)` before passing `text` to `IsChatMessage` or `IsPrivateMessage`.

<ParamField path="text" type="string" required>
  Plain-text representation of the message, including Minecraft color codes (`§`).
</ParamField>
<ParamField path="json" type="string?">
  Raw JSON string from the server. `null` on MC 1.5 and older.
</ParamField>

---

### OnDisconnect

```csharp
public virtual bool OnDisconnect(DisconnectReason reason, string message)
```

Called when the client disconnects. Return `true` to signal that MCC is about to restart (for informational use; the restart decision is made by MCC itself).

<ParamField path="reason" type="DisconnectReason" required>
  One of `InGameKick`, `LoginRejected`, `ConnectionLost`, or `UserLogout`.
</ParamField>
<ParamField path="message" type="string" required>
  The kick or disconnect message, if any.
</ParamField>

---

### OnDeath / OnRespawn

```csharp
public virtual void OnDeath()
public virtual void OnRespawn()
```

`OnDeath` fires when the player dies. `OnRespawn` fires when the player respawns after death or after changing dimensions.

---

</ParamField>
<ParamField path="food" type="int" required>

</ParamField>

### OnPlayerJoin / OnPlayerLeave

</ParamField>
<ParamField path="name" type="string" required>

### OnTeam

```csharp
public virtual void OnTeam(string teamName, byte method, string displayName,
    byte friendlyFlags, string nameTagVisibility, string collisionRule,
    int color, string prefix, string suffix, List<string> players)
```

Called whenever a scoreboard team packet arrives.

<ParamField path="teamName" type="string" required>Internal team name (up to 16 characters).</ParamField>
<ParamField path="method" type="byte" required>
  `0` = create, `1` = remove, `2` = update parameters, `3` = add players, `4` = remove players.
</ParamField>
<ParamField path="displayName" type="string">Formatted display name. Present when `method` is 0 or 2.</ParamField>
<ParamField path="friendlyFlags" type="byte">Bit 0 = allowFriendlyFire, bit 1 = seeFriendlyInvisibles. Present when `method` is 0 or 2.</ParamField>
<ParamField path="nameTagVisibility" type="string">One of `"always"`, `"never"`, `"hideForOtherTeams"`, `"hideForOwnTeam"`. Present when `method` is 0 or 2.</ParamField>
<ParamField path="collisionRule" type="string">One of `"always"`, `"never"`, `"pushOtherTeams"`, `"pushOwnTeam"`. Present when `method` is 0 or 2.</ParamField>
<ParamField path="color" type="int">`ChatFormatting` ordinal. `-1` = none/reset. Present when `method` is 0 or 2.</ParamField>
<ParamField path="prefix" type="string">Member name prefix. Present when `method` is 0 or 2.</ParamField>
<ParamField path="suffix" type="string">Member name suffix. Present when `method` is 0 or 2.</ParamField>
<ParamField path="players" type="List&lt;string&gt;">Player or entity names. Present when `method` is 0, 3, or 4.</ParamField>

## Utility methods

These `protected` methods are available to every bot via the `ChatBot` toolbox.

### SendText

```csharp
protected bool SendText(string text, bool sendImmediately = false)
```

Sends a message or command to the server. Prefix with `/` to send a server-side command.

<ParamField path="text" type="string" required>Text or command to send. Max length: use `GetMaxChatMessageLength()` to check.</ParamField>
<ParamField path="sendImmediately" type="bool">Deprecated parameter kept for script compatibility. Ignored.</ParamField>

---

### LogToConsole / LogDebugToConsole

```csharp
protected void LogToConsole(object? text)
protected void LogDebugToConsole(object text)
```

Write a message to the MCC console. `LogDebugToConsole` only outputs when debug logging is enabled in MCC configuration. Neither method sends anything to the server.

---

### LoadBot / UnLoadBot

```csharp
protected void LoadBot(ChatBot bot)
protected void UnLoadBot(ChatBot bot)
```

Dynamically load or unload a bot instance at runtime. `LoadBot` first unloads any existing instance of the same bot before loading the new one.

---

### RunScript

```csharp
protected void RunScript(string filename, string? playername = null,
    Dictionary<string, object>? localVars = null)
```

Compile and load another `//MCCScript` file.

<ParamField path="filename" type="string" required>Path to the `.cs` script file.</ParamField>
<ParamField path="playername" type="string?">Player name to receive error messages, if applicable.</ParamField>
<ParamField path="localVars" type="Dictionary&lt;string,object&gt;?">Local variables to inject into the script.</ParamField>

---

### GetVerbatim

```csharp
public static string GetVerbatim(string? text)
```

Strips Minecraft color codes (`§c`, `§#rrggbb`) from a string. Always call this before passing server text to `IsChatMessage` or `IsPrivateMessage`.

---

### IsChatMessage / IsPrivateMessage

```csharp
protected static bool IsChatMessage(string text, ref string message, ref string sender)
protected static bool IsPrivateMessage(string text, ref string message, ref string sender)
```

Parse common chat formats from vanilla and popular server plugins. Both methods write the extracted `message` body and `sender` name into the `ref` parameters and return `true` on a successful parse. Pass verbatim (color-stripped) text.

---

### GetUsername / GetUserUUID

```csharp
protected string GetUsername()
protected string GetUserUUID()
```

### GetGamemode

```csharp
protected int GetGamemode()
```

### GetOnlinePlayers / GetOnlinePlayersWithUUID

```csharp
protected string[] GetOnlinePlayers()
```

### GetTimestamp / GetShortTimestamp

```csharp
protected static string GetTimestamp()
```

Return a formatted timestamp string. `GetTimestamp` returns `yyyy-MM-dd HH:mm:ss`; `GetShortTimestamp` returns `HH:mm:ss`.

### GetServerTPS


Returns the most recent server TPS estimate calculated by MCC from world time update packets.

### GetProtocolVersion


### ScheduleOnMainThread

```csharp
protected void ScheduleOnMainThread(Action task, int delayTicks = 0)
protected void ScheduleOnMainThread(Action task, TimeSpan delay)
```

Schedule an action to run on the MCC main thread after an optional delay. Does not block the caller. One tick is approximately 50 ms at 20 TPS.

<ParamField path="task" type="Action" required>The action to run.</ParamField>
<ParamField path="delayTicks" type="int">Number of ticks to wait before running. `0` means run as soon as possible.</ParamField>
<ParamField path="delay" type="TimeSpan">Alternatively, specify a `TimeSpan` delay.</ParamField>

---

### InvokeOnMainThread

```csharp
protected T InvokeOnMainThread<T>(Func<T> task)
protected void InvokeOnMainThread(Action task)
```

Run a task on the MCC main thread and **wait** for it to complete. Use this to safely read or mutate shared state from a background thread.

---

### ReconnectToTheServer / DisconnectAndExit

```csharp
protected void ReconnectToTheServer(int ExtraAttempts = 3, int delaySeconds = 0,
    bool keepAccountAndServerSettings = false)
protected void DisconnectAndExit()
```

`ReconnectToTheServer` triggers a full reconnect cycle. `DisconnectAndExit` disconnects and exits MCC entirely.

---

### SendPrivateMessage

```csharp
protected void SendPrivateMessage(string player, string message)
```

Sends a private message using the configured private-message command (default `/msg`).

### RegisterChatBotCommand

```csharp
protected void RegisterChatBotCommand(string cmdName, string cmdDesc,
    string cmdUsage, CommandRunner callback)
```

Register a custom MCC internal command backed by Brigadier. The command is automatically unregistered when the bot is unloaded.

### App variables

## Complete example

The following script demonstrates the most common patterns: initialization, chat parsing, health monitoring, and a scheduled action.

```csharp
//MCCScript 1.0

MCC.LoadBot(new DemoBot());

//MCCScript Extensions

public class DemoBot : ChatBot
{
    private int tickCount = 0;

    public override void Initialize()
    {
        // Safe to read config or set up state here.
        // Do NOT call SendText here.
        LogToConsole("DemoBot loaded.");
    }

    public override void AfterGameJoined()
    {
        // Safe to send chat or commands after joining.
        LogToConsole("Joined as " + GetUsername() + " on " + GetServerHost());
        // Schedule a greeting 40 ticks (~2 seconds) after joining.
        ScheduleOnMainThread(() => SendText("Hello, world!"), 40);
    }

    public override void Update()
    {
        tickCount++;
        if (tickCount >= 200) // every ~10 seconds
        {
            tickCount = 0;
            LogDebugToConsole("Tick heartbeat");
        }
    }

    public override void GetText(string text)
    {
        string message = "";
        string sender = "";
        text = GetVerbatim(text);

        if (IsChatMessage(text, ref message, ref sender))
        {
            if (message.ToLower() == "ping")
                SendText("Pong!");
        }
        else if (IsPrivateMessage(text, ref message, ref sender))
        {
            LogToConsole("PM from " + sender + ": " + message);
        }
    }

    public override void OnHealthUpdate(float health, int food)
    {
        if (health <= 6.0f)
            LogToConsole("Warning: low health (" + health + ")");
    }

    public override bool OnDisconnect(DisconnectReason reason, string message)
    {
        LogToConsole("Disconnected: " + reason + " - " + message);
        return false;
    }
}
```
