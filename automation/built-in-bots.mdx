---
title: "Built-in Automation Bots for MCC"
description: "MCC includes 28 built-in bots for AFK prevention, fishing, farming, Discord and Telegram bridges, remote control, and more. Enable each in the config file."
---

Minecraft Console Client ships with 28 built-in automation bots called Chat Bots. Each bot is configured in the `MinecraftClient.ini` config file under its own `[ChatBot.BotName]` section. To enable a bot, open the config file, locate the relevant section, and set `Enabled = true`. Most bots are disabled by default and will not run until you explicitly turn them on.

<Note>
All settings referenced on this page live in the MCC configuration file. The section header for each bot is shown as `[ChatBot.BotName]`. Some bots require additional MCC features such as `TerrainAndMovements`, `InventoryHandling`, or `EntityHandling` to be enabled before they will work.
</Note>

## Chat & Alerts

<AccordionGroup>
  <Accordion title="Alerts">
    Get alerted when specified words appear in chat. Useful for moderation or detecting when someone mentions your name.

    **Config section:** `[ChatBot.Alerts]`

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Beep_Enabled` | boolean | `true` | Play a beep sound on alert (system-dependent) |
    | `Trigger_By_Words` | boolean | `false` | Alert when a keyword from `Matches` is detected |
    | `Trigger_By_Rain` | boolean | `false` | Alert when rain starts or stops |
    | `Trigger_By_Thunderstorm` | boolean | `false` | Alert at the start and end of thunderstorms |
    | `Log_To_File` | boolean | `false` | Write alerts to a file |
    | `Log_File` | string | `"alerts-log.txt"` | Path to the alerts log file |
    | `Matches` | array of strings | `[]` | Words that trigger an alert |
    | `Excludes` | array of strings | `[]` | Words that suppress an alert |

    ```toml
    [ChatBot.Alerts]
    Enabled = true
    Trigger_By_Words = true
    Matches = [ "Yourname", " whispers ", "-> me", "admin" ]
    Excludes = [ "myserver.com", "Yourname>:" ]
    ```
  </Accordion>

  <Accordion title="AutoRespond">
    Run commands or send messages automatically when a specified pattern is detected in chat.

    **Config section:** `[ChatBot.AutoRespond]`

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Matches_File` | string | `"matches.ini"` | Path to the file containing match/response rules |
    | `Match_Colors` | boolean | `false` | Preserve color formatting when matching |

    <Warning>
      Server admins can spoof private messages using `/tellraw` or `/nick`. Enable AutoRespond only on servers you trust. The bot can also become spammy; use the global `messagecooldown` setting to limit outgoing messages.
    </Warning>

    Rules are defined in an external file (default `matches.ini`). See the [`sample-matches.ini`](https://github.com/MCCTeam/Minecraft-Console-Client/blob/master/MinecraftClient/config/sample-matches.ini) for a detailed format guide.

    ```toml
    [ChatBot.AutoRespond]
    Enabled = true
    Matches_File = "matches.ini"
    ```
  </Accordion>

  <Accordion title="ChatLog">
    Log all chat messages to a file, with optional timestamps and message-type filtering.

    **Config section:** `[ChatBot.ChatLog]`

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Add_DateTime` | boolean | `true` | Prepend timestamps to logged messages |
    | `Log_File` | string | `"chatlog-%username%-%serverip%.txt"` | Log file name (supports variables) |
    | `Filter` | string | `"messages"` | What to log: `all`, `messages`, `chat`, `private`, or `internal` |

    ```toml
    [ChatBot.ChatLog]
    Enabled = true
    Add_DateTime = true
    Filter = "messages"
    ```
  </Accordion>
</AccordionGroup>

## AFK & Reconnect

<AccordionGroup>
  <Accordion title="AntiAFK">
    Prevent automatic AFK disconnection by sending a command, sneaking, or walking randomly on a configurable interval.

    **Config section:** `[ChatBot.AntiAFK]`

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Delay` | inline table | `{ min = 60.0, max = 60.0 }` | Interval in seconds between actions; set different min/max for randomness |
    | `Command` | string | `"/ping"` | Command to send on each trigger |
    | `Use_Sneak` | boolean | `false` | Sneak on each trigger |
    | `Use_Terrain_Handling` | boolean | `false` | Walk randomly instead of sending a command |
    | `Walk_Range` | integer | `5` | Range (in blocks) for random walking (-X to +X and -Z to +Z) |
    | `Walk_Retries` | integer | `20` | Path attempts before falling back to command method |

    <Note>
      `Use_Terrain_Handling` requires `TerrainAndMovements` to be enabled. Enclose the bot in a small area (recommended 5x5x5) to prevent it from wandering off.
    </Note>

    ```toml
    [ChatBot.AntiAFK]
    Enabled = true
    Delay = { min = 55.0, max = 70.0 }
    Command = "/ping"
    ```
  </Accordion>

  <Accordion title="AutoRelog">
    Automatically reconnect to the server when disconnected, with configurable delay, retry limit, and kick-message filtering.

    **Config section:** `[ChatBot.AutoRelog]`

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Delay` | inline table | `{ min = 3.0, max = 3.0 }` | Delay before reconnecting in seconds |
    | `Retries` | integer | `-1` | Number of reconnect attempts; `-1` for infinite |
    | `Ignore_Kick_Message` | boolean | `false` | Reconnect regardless of kick message content |
    | `Kick_Messages` | array of strings | `[ "Connection has been lost", "Server is restarting", ... ]` | Kick message keywords that trigger reconnect |

    <Warning>
      Infinite retries (`Retries = -1`) may get you banned from some servers if the server detects repeated reconnections.
    </Warning>

    ```toml
    [ChatBot.AutoRelog]
    Enabled = true
    Delay = { min = 5.0, max = 10.0 }
    Retries = 10
    Kick_Messages = [ "Server is restarting", "Server is full" ]
    ```
  </Accordion>
</AccordionGroup>

## Gameplay Automation

<AccordionGroup>
  <Accordion title="AutoAttack">
    Automatically attack mobs in range. Supports single or multi-target mode, priority by health or distance, and a whitelist/blacklist of entity types.

    **Config section:** `[ChatBot.AutoAttack]`

    <Note>
      Requires `InventoryHandling` and `EntityHandling` to be enabled.
    </Note>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Mode` | string | `"single"` | `single` (one mob per swing) or `multi` (all in range) |
    | `Priority` | string | `"distance"` | Target priority: `health` or `distance` |
    | `Cooldown_Time` | inline table | `{ Custom = false, value = 1.0 }` | Custom attack cooldown in seconds |
    | `Interaction` | string | `"Attack"` | `Attack`, `Interact`, or `InteractAt` |
    | `Attack_Hostile` | boolean | `true` | Attack hostile mobs |
    | `Attack_Passive` | boolean | `false` | Attack passive mobs |
    | `List_Mode` | string | `"whitelist"` | `whitelist` (only attack listed) or `blacklist` (skip listed) |
    | `Entites_List` | array of strings | `[ "Zombie", "Cow" ]` | Entity types to whitelist or blacklist |

    ```toml
    [ChatBot.AutoAttack]
    Enabled = true
    Mode = "single"
    Attack_Hostile = true
    Attack_Passive = false
    List_Mode = "whitelist"
    Entites_List = [ "Zombie", "Skeleton", "Spider" ]
    ```
  </Accordion>

  <Accordion title="AutoCraft">
    Automatically craft items using recipes defined in the config. Supports both 2x2 player inventory crafting and 3x3 crafting table crafting.

    **Config section:** `[ChatBot.AutoCraft]`

    <Note>
      Requires `InventoryHandling`. For crafting table recipes, `TerrainAndMovements` must also be enabled.
    </Note>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `CraftingTable` | inline table | â€” | Coordinates of the crafting table: `{ X = 0.0, Y = 0.0, Z = 0.0 }` |
    | `OnFailure` | string | `"abort"` | What to do when crafting fails: `abort` or `wait` |

    Recipes are defined as `[[ChatBot.AutoCraft.Recipes]]` subsections. Each recipe has a `Name`, `Type` (`player` or `table`), `Result` item type, and `Slots` array.

    **Commands:** `/autocraft list`, `/autocraft start <name>`, `/autocraft stop`, `/autocraft help`

    ```toml
    [ChatBot.AutoCraft]
    Enabled = true
    CraftingTable = { X = 123.0, Y = 65.0, Z = 456.0 }
    OnFailure = "wait"

    [[ChatBot.AutoCraft.Recipes]]
    Name = "stone-bricks"
    Type = "player"
    Result = "StoneBricks"
    Slots = [ "Stone", "Stone", "Stone", "Stone" ]
    ```
  </Accordion>

  <Accordion title="AutoDig">
    Automatically dig blocks at specified locations or at the block the bot is looking at.

    **Config section:** `[ChatBot.AutoDig]`

    <Note>
      Requires `InventoryHandling` and `TerrainAndMovements`.
    </Note>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Mode` | string | `"lookat"` | `lookat`, `fixedpos`, or `both` |
    | `Locations` | array of inline tables | `[]` | List of `{ x, y, z }` coordinates to dig |
    | `Location_Order` | string | `"distance"` | `distance` (nearest first) or `index` (list order) |
    | `Auto_Start_Delay` | float | `3.0` | Seconds to wait before auto-starting; `-1` to disable |
    | `Auto_Tool_Switch` | boolean | `false` | Switch to the best tool automatically |
    | `Durability_Limit` | integer | `2` | Skip tools below this durability |
    | `List_Type` | string | `"whitelist"` | `whitelist` or `blacklist` for the `Blocks` list |
    | `Blocks` | array of strings | `[ "Cobblestone", "Stone" ]` | Block types to dig or skip |

    **Commands:** `/digbot start`, `/digbot stop`

    ```toml
    [ChatBot.AutoDig]
    Enabled = true
    Mode = "fixedpos"
    Locations = [
      { x = 123.5, y = 64.0, z = 234.5 },
    ]
    ```
  </Accordion>

  <Accordion title="AutoDrop">

  </Accordion>

  <Accordion title="AutoEat">

    | Setting | Type | Default | Description |
    |---|---|---|---|

  <Accordion title="AutoFishing">
    Automatically cast and reel in a fishing rod. Bite detection uses bobber movement, velocity, and splash sounds for high reliability.

    **Config section:** `[ChatBot.AutoFishing]`

    <Note>
      Requires `EntityHandling`. For automatic rod switching, also enable `InventoryHandling`. For position adjustment after each catch, enable `TerrainAndMovements`.
    </Note>

    <Tip>
      A fishing rod with the **Mending** enchantment is strongly recommended so it repairs itself from fishing XP.
    </Tip>

    **Quick start:** Hold a fishing rod aimed at water before logging in with MCC, then enable `AutoFishing` in the config. You will see `Fishing will start in 3.0 second(s).` in the console.

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Auto_Start` | boolean | `true` | Start fishing automatically after joining |
    | `Cast_Delay` | float | `0.4` | Seconds to wait before recasting after a catch |
    | `Fishing_Delay` | float | `3.0` | Seconds to wait after joining before starting |
    | `Fishing_Timeout` | float | `300.0` | Seconds without a bite before recasting |
    | `Durability_Limit` | integer | `2` | Skip rods with less than this durability |
    | `Auto_Rod_Switch` | boolean | `true` | Switch to a new rod when the current one breaks |
    | `Enable_Velocity_Detection` | boolean | `true` | Use velocity packets for bite detection |
    | `Enable_Sound_Detection` | boolean | `true` | Use splash sounds for bite detection |
    | `Enable_Move` | boolean | `false` | Change position/angle after each catch |

    ```toml
    [ChatBot.AutoFishing]
    Enabled = true
    Auto_Start = true
    Auto_Rod_Switch = true
    Durability_Limit = 5
    ```
  </Accordion>

  <Accordion title="Farmer">
    Plant, break, and bonemeal crops automatically. Supports beetroot, carrot, melon, netherwart, pumpkin, potato, and wheat.

    **Config section:** `[ChatBot.Farmer]`

    <Note>
      Requires `TerrainAndMovements` and `InventoryHandling`.
    </Note>

    <Warning>
      This bot is experimental. It was primarily tested on 1.19.2. Some edge cases remain such as missed bonemealing and rare farmland-breaking. Keep farming areas flat and fenced.
    </Warning>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Delay_Between_Tasks` | integer | `1` | Seconds between each task (minimum 1) |

    **Commands:**
    ```
    /farmer start <crop type> [radius:<blocks>] [unsafe:<true/false>] [teleport:<true/false>] [debug:<true/false>]
    /farmer stop
    ```

    ```toml
    [ChatBot.Farmer]
    Enabled = true
    Delay_Between_Tasks = 1
    ```

    Then in-game: `/farmer start wheat radius:40`
  </Accordion>

  <Accordion title="FollowPlayer">
    Follow a specific player around the world using pathfinding.

    **Config section:** `[ChatBot.FollowPlayer]`

    <Note>

    </Note>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Update_Limit` | float | `1.5` | Calculation rate in seconds |
    | `Stop_At_Distance` | float | `3.0` | Stop following when within this many blocks |

    ```toml
    [ChatBot.FollowPlayer]
    Enabled = true
    Stop_At_Distance = 3.0
    ```
  </Accordion>

  <Accordion title="ItemsCollector">
    Walk to and collect dropped items on the ground within a configurable radius.

    | Setting | Type | Default | Description |

## Remote Access & Bridges

<AccordionGroup>

    | `Authorized_Chat_Ids` | array of strings | `[]` | Chat IDs allowed to send commands |
    | `Message_Send_Timeout` | integer | `3` | Seconds to wait before cancelling a send |
    | `PrivateMessageFormat` | string | `"*(Private Message)* {username}: {message}"` | Format for private message notifications |
    | `PublicMessageFormat` | string | `"{username}: {message}"` | Format for public chat |

    ```toml
    [ChatBot.TelegramBridge]
    Enabled = true
    Token = "your-telegram-bot-token"
    ChannelId = "2627844670"
    Authorized_Chat_Ids = [ "2627844670" ]
    ```
  </Accordion>

  <Accordion title="RemoteControl">
    Execute MCC console commands by sending private messages to the bot in-game via `/tell`.

    **Config section:** `[ChatBot.RemoteControl]`

    You must configure `ChatFormat` correctly and add yourself to `botowners` in the MCC config for the bot to recognize your commands.

    <Warning>
      Server admins can spoof private messages using `/tellraw` or `/nick`. Enable this only on servers you trust.
    </Warning>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `AutoTpaccept` | boolean | `true` | Automatically accept teleport requests from bot owners |
    | `AutoTpaccept_Everyone` | boolean | `false` | Automatically accept teleport requests from all players |

    ```toml
    [ChatBot.RemoteControl]
    Enabled = true
    AutoTpaccept = true
    AutoTpaccept_Everyone = false
    ```
  </Accordion>

  <Accordion title="McpServer">
    Expose a local MCP (Model Context Protocol) HTTP endpoint so AI agents such as Claude Code, OpenCode, Codex, and Cursor can control MCC directly.

    **Config section:** `[ChatBot.McpServer]`

    The MCP server starts after MCC joins the game and listens on `http://127.0.0.1:33333/mcp` by default. AI agents connect to this endpoint to read chat, check players, move around, inspect inventories, and interact with entities.

    <Warning>
      This feature is experimental and still being tested and polished. Secure it with an auth token even for local use.
    </Warning>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Transport.BindHost` | string | `"127.0.0.1"` | IP to bind; use `0.0.0.0` to expose to the network |
    | `Transport.Port` | integer | `33333` | TCP port for the HTTP endpoint |
    | `Transport.Route` | string | `"/mcp"` | HTTP path for the MCP endpoint |
    | `Transport.RequireAuthToken` | boolean | `false` | Require a bearer token (recommended: `true`) |
    | `Transport.AuthTokenEnvVar` | string | `"MCC_MCP_AUTH_TOKEN"` | Environment variable holding the token |
    | `Capabilities.SessionStatus` | boolean | `true` | Session, server, chat-history, and status tools |
    | `Capabilities.ChatAndCommands` | boolean | `true` | Chat, commands, respawn, and direct-control tools |
    | `Capabilities.Movement` | boolean | `true` | Pathfinding, movement, and nearby-block query tools |
    | `Capabilities.Inventory` | boolean | `true` | Inventory, container, and item management tools |
    | `Capabilities.EntityWorld` | boolean | `true` | Entity queries, sign search, and dropped-item tools |

    ```toml
    [ChatBot.McpServer]
    Enabled = true
    Transport = { BindHost = "127.0.0.1", Port = 33333, Route = "/mcp", RequireAuthToken = true, AuthTokenEnvVar = "MCC_MCP_AUTH_TOKEN" }
    Capabilities = { SessionStatus = true, ChatAndCommands = true, Movement = true, Inventory = true, EntityWorld = true }
    ```

    ```bash
    export MCC_MCP_AUTH_TOKEN="your-secret-token"
    ```
  </Accordion>
</AccordionGroup>

## Recording & Scheduling

<AccordionGroup>
  <Accordion title="ReplayCapture">
    Record gameplay sessions and play them back later using the [Replay Mod](https://www.replaymod.com/).

    **Config section:** `[ChatBot.ReplayCapture]`

    <Warning>
      Always stop recording with `/replay stop` or exit MCC gracefully with `/quit`. Closing the application forcefully may corrupt the replay file. This bot does not work on Minecraft 1.19.
    </Warning>

    <Note>
      Due to technical limitations, the MCC client player will not appear in the replay file.
    </Note>

    | Setting | Type | Default | Description |
    |---|---|---|---|
    | `Enabled` | boolean | `false` | Enable or disable the bot |
    | `Backup_Interval` | float | `300.0` | Seconds between auto-saves; `-1` to disable |

    **Commands:** `/replay start`, `/replay stop`

    ```toml
    [ChatBot.ReplayCapture]
    Enabled = true
    Backup_Interval = 300.0
    ```
  </Accordion>

  <Accordion title="ScriptScheduler">
    Schedule MCC commands and scripts to run on server join, at specific times of day, or on a repeating interval.

    **Config section:** `[ChatBot.ScriptScheduler]`

    Each scheduled task is defined as a `[[ChatBot.ScriptScheduler.TaskList]]` subsection.

    | Setting | Type | Description |
    |---|---|---|
    | `Task_Name` | string | Human-readable name for the task |
    | `Trigger_On_First_Login` | boolean | Run once on the first login only |
    | `Trigger_On_Login` | boolean | Run on every login |
    | `Trigger_On_Times` | inline table | Run at specific times: `{ Enable = true, Times = [ 14:00:00 ] }` |
    | `Trigger_On_Interval` | inline table | Run on a recurring interval: `{ Enable = true, MinTime = 30.0, MaxTime = 60.0 }` |
    | `Action` | string | The MCC command or script to run |

    ```toml
    [ChatBot.ScriptScheduler]
    Enabled = true

        [[ChatBot.ScriptScheduler.TaskList]]
        Task_Name = "Login command"
        Trigger_On_First_Login = false
        Trigger_On_Login = true
        Trigger_On_Times = { Enable = false, Times = [] }
        Trigger_On_Interval = { Enable = false, MinTime = 1.0, MaxTime = 10.0 }
        Action = "send /login mypassword"

        [[ChatBot.ScriptScheduler.TaskList]]
        Task_Name = "Daily announcement"
        Trigger_On_First_Login = false
        Trigger_On_Login = false
        Trigger_On_Times = { Enable = true, Times = [ 14:00:00 ] }
        Trigger_On_Interval = { Enable = false, MinTime = 1.0, MaxTime = 1.0 }
        Action = "send Good afternoon everyone!"
    ```
  </Accordion>
</AccordionGroup>
